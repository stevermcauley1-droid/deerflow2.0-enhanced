# 多语言翻译技能 (Multi-Language Translation)

专业化的多语言翻译技能，支持技术文档、学术论文、商业报告的精准翻译。

## 概述

本技能提供高质量的多语言翻译：
- 支持多种语言对互译
- 专业术语库支持
- 格式保持一致
- 上下文理解
- 翻译记忆

## 支持的语言

| 语言 | 代码 | 备注 |
|------|------|------|
| 中文 | zh | 简体/繁体 |
| 英语 | en | 美式/英式 |
| 日语 | ja | 商务/口语 |
| 德语 | de | 专业术语 |
| 法语 | fr | 法国/加拿大 |
| 韩语 | ko |  |
| 西班牙语 | es | 拉美/西班牙 |

## 核心能力

### 1. 文档翻译
- Markdown 文档
- 技术文档
- 学术论文
- 商业报告
- 网站内容
- 产品描述

### 2. 术语管理
- 行业术语库
- 专有名词保持
- 缩写首次翻译

### 3. 格式保持
- 标题层级
- 表格结构
- 代码块
- 链接/引用
- 列表格式

### 4. 语境理解
- 理解专业背景
- 保持语气一致
- 文化适配
- 本地化优化

## 工作流程

### Step 1: 确认需求

1. 源语言是什么
2. 目标语言是什么
3. 文档类型（技术/商业/学术）
4. 用途（发布/参考/内部）
5. 特殊要求（术语/风格）

### Step 2: 分析文档

1. 识别文档类型
2. 提取术语列表
3. 识别格式元素
4. 评估难度/专业度

### Step 3: 翻译执行

**原则：**
- 准确性 > 流利性
- 专业术语统一
- 保持原文结构
- 适当本地化

### Step 4: 质量检查

1. 术语一致性
2. 格式正确性
3. 语法准确性
4. 表达自然度
5. 术语统一性

### Step 5: 输出

提供：
- 翻译后的文档
- 术语表（如需要）
- 格式对比（可选）

## 翻译标准

### 术语标准
```
首次出现：英文原文 (中文翻译)
后续出现：仅中文翻译

示例：
Large Language Model (LLM, 大语言模型)
LLM 是一种...
```

### 格式标准
```markdown
# 标题 → # Title
## 副标题 → ## Subtitle

- 列表项 → - List item
1. 有序列表 → 1. Ordered list

| 表格 | 表格 |
|-----|-----|
| 内容 | Content |

`代码` → `code`

> 引用 → > Quote
```

### 标点标准
- 中文文档：全角标点（，。：；？！""）
- 英文文档：半角标点（,.:;?!""）
- 中英混排：按主要语言选择

### 翻译检查清单
- [ ] 术语一致性

### 常见错误预防
1. 过度直译 → 结合语境
2. 术语不一致 → 建立术语表
3. 格式丢失 → 逐元素检查
4. 链接失效 → 验证链接

## 注意事项

1. **不添加解释** - 翻译是翻译，不是解释
2. **保持原意** - 不改变作者观点
3. **术语一致** - 建立并使用术语表
4. **格式优先** - 先保格式，再保内容
5. **专业求证** - 不确定的术语查证后再翻

## 输出

1. 翻译后的文档（Markdown）
2. 可选：HTML 版本
3. 可选：术语表
